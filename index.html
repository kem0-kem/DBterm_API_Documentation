1. 시스템 소개

DBterm API는 후원금의 수집부터 집행까지 전 과정을 데이터베이스에 기록하고, 역할별로 검증 가능한 형태로 조회할 수 있게 하는 백엔드 API입니다.

후원자는 자신의 후원 내역과 실제 집행 정보를 추적할 수 있고, 운영자와 회계 담당자는 캠페인과 배분·지급 절차를 관리하며, 감사는 로그와 불일치 내역을 점검하여 투명성을 보장합니다.

2. 인증 방식

모든 보호된 API는 인증 토큰 기반으로 호출합니다. 로그인 성공 시 발급되는 액세스 토큰을 이후 요청의 Authorization 헤더에 포함해야 합니다. 

Authorization: Bearer <access-token>

3. 역할 및 권한

시스템에는 다음과 같은 주요 역할이 존재합니다.

Admin Donor Operator Accountant Auditor
역할 주요 권한
Admin 사용자 계정 전체 관리, 역할 부여 및 변경
Donor 본인 후원 내역 조회, 후원금 사용처 추적
Operator 캠페인 생성·수정, 기부 검증, 배분안 검토 및 승인
Accountant 승인된 배분에 대한 지급 처리, 수혜자 정보 관리, 증빙 업로드
Auditor 감사 로그 조회, 배분·지급 불일치 탐지, 문서 해시 검증

4. Auth API

아이디와 비밀번호를 이용해 로그인하고 액세스 토큰을 발급받습니다.

요청 예시 (YAML)
body:
  username: "donor01"
  password: "passw0rd!"
응답 예시 (YAML)
status: 200
body:
  accessToken: "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  user:
    userId: 1
    username: "donor01"
    role: "DONOR"

현재 로그인한 사용자의 기본 정보를 반환합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
응답 예시 (YAML)
status: 200
body:
  userId: 1
  username: "donor01"
  displayName: "홍길동"
  role: "DONOR"
  anonymous: false

표시 이름, 익명 여부 등 일부 프로필 정보를 수정합니다. 모든 변경 내역은 감사 로그에 기록됩니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
body:
  displayName: "익명 후원자"
  anonymous: true
응답 예시 (YAML)
status: 200
body:
  userId: 1
  username: "donor01"
  displayName: "익명 후원자"
  role: "DONOR"
  anonymous: true

5. Donor API

활성화된 후원 캠페인 목록을 조회합니다. Donor는 읽기 전용입니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
query:
  status: "ACTIVE"
  from: "2025-01-01"
  to: "2025-12-31"
응답 예시 (YAML)
status: 200
body:
  items:
    - campaignId: 101
      title: "소아암 환아 치료비 지원"
      description: "소아암 환아를 위한 치료비를 지원합니다."
      status: "ACTIVE"
      startDate: "2025-01-01"
      endDate: "2025-12-31"
      createdBy: 10
    - campaignId: 102
      title: "저소득층 장학금"
      description: "저소득층 아동을 위한 장학금 캠페인입니다."
      status: "ACTIVE"
      startDate: "2025-03-01"
      endDate: "2025-09-30"

로그인한 후원자의 기부 내역을 기간 및 캠페인별로 조회합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
query:
  from: "2025-01-01"
  to: "2025-12-31"
  campaignId: 101
  sort: "DATE_DESC"
응답 예시 (YAML)
status: 200
body:
  items:
    - donationId: 5001
      campaignId: 101
      amount: 30000
      donatedAt: "2025-05-10T12:30:00Z"
      verified: true
    - donationId: 5002
      campaignId: 101
      amount: 20000
      donatedAt: "2025-08-01T09:15:00Z"
      verified: true

선택한 후원 건이 어떤 수혜자에게 얼마씩 배분되고, 실제로 언제 지급되었는지 배분과 지급 정보를 함께 조회합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
path:
  donationId: 5001
응답 예시 (YAML)
status: 200
body:
  donation:
    donationId: 5001
    campaignId: 101
    amount: 30000
    donatedAt: "2025-05-10T12:30:00Z"
  allocations:
    - allocationId: 9001
      receiverId: 201
      receiverName: "김아름"
      amount: 20000
      status: "EXECUTED"
      disbursements:
        - disbursementId: 7001
          amount: 20000
          executedAt: "2025-05-20T09:00:00Z"
    - allocationId: 9002
      receiverId: 202
      receiverName: "이민수"
      amount: 10000
      status: "APPROVED"
      disbursements: []

6. Operator API

새로운 후원 캠페인을 등록합니다. 생성자는 자동으로 createdBy에 기록됩니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
body:
  title: "소아암 환아 치료비 지원"
  description: "소아암 환아를 위한 치료비 지원 캠페인입니다."
  startDate: "2025-01-01"
  endDate: "2025-12-31"
  status: "ACTIVE"
응답 예시 (YAML)
status: 201
body:
  campaignId: 101
  title: "소아암 환아 치료비 지원"
  description: "소아암 환아를 위한 치료비 지원 캠페인입니다."
  startDate: "2025-01-01"
  endDate: "2025-12-31"
  status: "ACTIVE"
  createdBy: 10

오늘 들어온 미검증 기부 건을 중심으로, 캠페인별·금액순으로 필터링하여 조회합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
query:
  verified: false
  date: "2025-05-10"
응답 예시 (YAML)
status: 200
body:
  items:
    - donationId: 6001
      campaignId: 101
      donorUsername: "donor02"
      amount: 50000
      donatedAt: "2025-05-10T10:00:00Z"
      verified: false

결제 모듈에서 생성된 기부 건의 verified 상태를 변경합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
path:
  donationId: 6001
body:
  verified: true
응답 예시 (YAML)
status: 200
body:
  donationId: 6001
  campaignId: 101
  amount: 50000
  verified: true

자동 또는 수동으로 생성된 배분안을 조회하고, 상태별로 필터링합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
query:
  status: "PENDING"
  campaignId: 101
응답 예시 (YAML)
status: 200
body:
  items:
    - allocationId: 9001
      donationId: 5001
      receiverId: 201
      receiverName: "김아름"
      amount: 20000
      status: "PENDING"

배분 금액과 수혜자를 조정하고, PENDING 상태의 배분안을 APPROVED 또는 REJECTED로 변경합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
path:
  allocationId: 9001
body:
  receiverId: 201
  amount: 20000
  status: "APPROVED"
응답 예시 (YAML)
status: 200
body:
  allocationId: 9001
  donationId: 5001
  receiverId: 201
  amount: 20000
  status: "APPROVED"

7. Accountant API

승인된 배분안 목록을 조회하여 실제 지급 작업을 준비합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
query:
  status: "APPROVED"
응답 예시 (YAML)
status: 200
body:
  items:
    - allocationId: 9001
      donationId: 5001
      receiverId: 201
      amount: 20000
      status: "APPROVED"

지정된 배분에 대해 지급을 기록하고, 같은 트랜잭션 안에서 배분 상태를 EXECUTED로 변경합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
body:
  allocationId: 9001
  amount: 20000
  paymentTxRef: "PG20250520001"
응답 예시 (YAML)
status: 201
body:
  disbursementId: 7001
  allocationId: 9001
  amount: 20000
  executedAt: "2025-05-20T09:00:00Z"
  paymentTxRef: "PG20250520001"
  allocationStatus: "EXECUTED"

지급 건에 대한 영수증, 증빙 문서 파일을 업로드하고 파일 경로 및 해시를 저장합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
path:
  disbursementId: 7001
body:
  fileName: "receipt-7001.pdf"
  storagePath: "s3://bucket/receipt-7001.pdf"
  fileHash: "b1946ac92492d2347c6235b4d2611184"
응답 예시 (YAML)
status: 201
body:
  documentId: 8001
  disbursementId: 7001
  fileName: "receipt-7001.pdf"
  storagePath: "s3://bucket/receipt-7001.pdf"
  fileHash: "b1946ac92492d2347c6235b4d2611184"
  uploadedAt: "2025-05-20T09:05:00Z"

8. Auditor API

사용자, 테이블, 액션별로 필터링 가능한 감사 로그를 조회합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
query:
  from: "2025-05-01"
  to: "2025-05-31"
  actorUserId: 10
  objectTable: "allocation"
응답 예시 (YAML)
status: 200
body:
  items:
    - auditId: 10001
      occurredAt: "2025-05-10T09:00:00Z"
      actorUserId: 10
      action: "UPDATE"
      objectTable: "allocation"
      objectId: "9001"
      detail: "status: PENDING -> APPROVED"

배분 금액과 실제 지급 금액이 일치하지 않는 건을 조회하여 추가 점검이 필요하도록 합니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
query:
  campaignId: 101
응답 예시 (YAML)
status: 200
body:
  items:
    - allocationId: 9100
      expectedAmount: 30000
      totalDisbursed: 25000
      diff: 5000
      status: "PARTIAL"

지급 증빙 문서의 해시 목록을 조회하여 외부 시스템이나 블록체인에 저장된 값과 대조할 수 있습니다.

요청 예시 (YAML)
headers:
  Authorization: "Bearer <access-token>"
query:
  from: "2025-05-01"
  to: "2025-05-31"
응답 예시 (YAML)
status: 200
body:
  items:
    - documentId: 8001
      disbursementId: 7001
      fileHash: "b1946ac92492d2347c6235b4d2611184"
      uploadedAt: "2025-05-20T09:05:00Z"
