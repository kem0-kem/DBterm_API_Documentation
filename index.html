1. 시스템 소개

DBterm API는 후원금의 수집부터 집행까지 전 과정을 데이터베이스에 기록하고, 역할별로 검증 가능한 형태로 조회할 수 있게 하는 백엔드 API입니다.

후원자는 자신의 후원 내역과 실제 집행 정보를 추적할 수 있고, 운영자와 회계 담당자는 캠페인과 배분·지급 절차를 관리하며, 감사는 로그와 불일치 내역을 점검하여 투명성을 보장합니다.

2. 인증 방식

모든 보호된 API는 인증 토큰 기반으로 호출합니다. 로그인 성공 시 발급되는 액세스 토큰을 이후 요청의 Authorization 헤더에 포함해야 합니다. 

{
  "Authorization": "Bearer <access-token>"
}

3. 역할 및 권한

시스템에는 다음과 같은 주요 역할이 존재합니다.

Admin Donor Operator Accountant Auditor
역할 주요 권한
Admin 사용자 계정 전체 관리, 역할 부여 및 변경
Donor 본인 후원 내역 조회, 후원금 사용처 추적
Operator 캠페인 생성·수정, 기부 건 검증, 배분안 검토 및 승인
Accountant 승인된 배분에 대한 지급 처리, 수혜자 정보 관리, 증빙 업로드
Auditor 감사 로그 조회, 배분·지급 불일치 탐지, 문서 해시 검증

4. Auth API

POST 로그인 /auth/login
Public

아이디와 비밀번호를 이용해 로그인하고 액세스 토큰을 발급받습니다.

Request Body 타입 필수 설명
username string Y 사용자 로그인 ID
password string Y 비밀번호(평문 또는 별도 암호화 규약에 따름)
Response 타입 설명
accessToken string API 호출에 사용하는 JWT 또는 토큰 값
user object userId, username, role 등 기본 정보
GET 내 정보 조회 /auth/me
Authenticated

현재 로그인한 사용자의 기본 정보를 반환합니다.

PUT 내 프로필 수정 /auth/me/profile
Authenticated

표시 이름, 익명 여부 등 일부 프로필 정보를 수정합니다. 모든 변경 내역은 감사 로그에 기록됩니다.

5. Donor API

GET 캠페인 목록 조회 /campaigns
Donor Operator

활성화된 후원 캠페인 목록을 조회합니다. Donor는 읽기 전용입니다.

Query 타입 필수 설명
status string N ACTIVE, INACTIVE 필터
from date N 시작일 기준 필터(예: 2025-01-01)
to date N 종료일 기준 필터
GET 내 후원 내역 조회 /donor/donations
Donor

로그인한 후원자의 기부 내역을 기간 및 캠페인별로 조회합니다.

Query 타입 필수 설명
from date N 조회 시작일
to date N 조회 종료일
campaignId number N 특정 캠페인만 필터링
sort string N DATE_ASC, DATE_DESC, AMOUNT_DESC 등
GET 후원금 사용처 추적 /donor/donations/{donationId}/trace
Donor

선택한 후원 건이 어떤 수혜자에게 얼마씩 배분되고, 실제로 언제 지급되었는지 배분(allocation)과 지급(disbursement) 정보를 함께 조회합니다.

6. Operator API

POST 캠페인 생성 /operator/campaigns
Operator

새로운 후원 캠페인을 등록합니다. 생성자는 자동으로 created_by에 기록됩니다.

GET 미검증 기부 조회 /operator/donations
Operator

오늘 들어온 미검증 기부 건을 중심으로, 캠페인별/금액순으로 필터링하여 조회합니다.

PATCH 기부 검증 상태 변경 /operator/donations/{donationId}/verify
Operator

결제 모듈에서 생성된 기부 건의 verified 상태를 변경합니다.

GET 배분안 목록 조회 /operator/allocations
Operator

자동 혹은 수동으로 생성된 배분안을 조회하고, 상태별로 필터링합니다.

PUT 배분안 수정 및 승인 /operator/allocations/{allocationId}
Operator

배분 금액과 수혜자를 조정하고, PENDING 상태의 배분안을 APPROVED 또는 REJECTED로 변경합니다.

7. Accountant API

GET 지급 대상 배분 조회 /accounting/allocations
Accountant

승인(APPROVED)된 배분안 목록을 조회하여 실제 지급 작업을 준비합니다.

POST 지급 기록 생성 /accounting/disbursements
Accountant

지정된 배분에 대해 지급(disbursement)을 기록하고, 같은 트랜잭션 안에서 배분 상태를 EXECUTED로 변경합니다.

POST 지급 증빙 업로드 /accounting/disbursements/{disbursementId}/documents
Accountant

지급 건에 대한 영수증, 증빙 문서 파일을 업로드하고 파일 경로 및 해시를 저장합니다.

8. Auditor API

GET 감사 로그 조회 /audits/logs
Auditor

사용자, 테이블, 액션별로 필터링 가능한 감사 로그를 조회합니다.

GET 배분·지급 불일치 목록 /audits/disbursement-mismatches
Auditor

배분 금액과 실제 지급 금액이 일치하지 않는 건을 조회하여 추가 점검이 필요하도록 합니다.

GET 문서 해시 목록 조회 /audits/document-hashes
Auditor

지급 증빙 문서의 해시 목록을 조회하여 외부 시스템이나 블록체인에 저장된 값과 대조할 수 있습니다.
